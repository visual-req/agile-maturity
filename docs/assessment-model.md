# 评估模型说明

## 评估范围

本工具用于评估客户在以下两个方向上的成熟度：

- Agile / Scrum
- DevOps

支持三种评估模式：

- 双评估：同时评估 Agile 与 DevOps
- 仅 Agile / Scrum
- 仅 DevOps

## Agile / Scrum 评估维度

当前包含以下维度：

- Scrum 角色与职责
- 事件执行
- 工件透明度
- 协作文化
- 度量与改进

每个维度包含多道题，重点关注：

- 角色职责是否清晰
- Scrum 事件是否有效执行
- 工件是否透明、可追踪
- 团队是否具备跨职能协作文化
- 是否通过度量驱动持续改进

## DevOps 评估维度

当前包含以下维度：

- 代码管理
- 持续集成
- 测试自动化
- 制品管理
- 配置管理
- 环境一致性
- 发布治理
- 监控反馈与安全左移

每个维度包含多道题，重点关注：

- 代码流转与评审治理
- 构建、检查与流水线自动化
- 分层测试与门禁机制
- 制品归档、传递、追踪与回滚
- 配置集中管理、审计与回退
- 环境复制能力与环境漂移治理
- 发布风险控制与灰度机制
- 监控、日志、追踪与安全左移

## 评分方式

- 每道题评分范围：`1-5 分`
- 每道题配置权重
- 系统按权重计算维度得分
- 维度得分换算为百分制
- 域得分按所属维度平均计算
- 综合得分按 Agile / DevOps 域得分平均计算

## 等级划分

当前等级区间如下：

- `0-34`：初始
- `35-59`：发展中
- `60-79`：稳定
- `80-100`：优化

## 报告输出内容

系统会基于答卷输出：

- 综合得分
- Agile 得分
- DevOps 得分
- 综合等级
- 各维度得分与等级
- 优势与现状
- 关键风险
- 短期 / 中期 / 长期改进建议

## 模型维护建议

- 新增维度时，应同步更新：
  - `src/data/questionBank.ts`
  - `src/utils/scoring.ts`
  - 首页维度概览
  - 文档说明
- 调整题目数量后，应同步验证：
  - 完成度计算
  - 报告维度统计
  - 测试用例
